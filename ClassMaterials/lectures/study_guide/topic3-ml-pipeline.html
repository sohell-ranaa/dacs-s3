Topic 3: Machine Learning Pipeline

The complete workflow from problem definition to model validation

Pipeline Overview

What is the ML Pipeline?

Machine learning (ML) pipeline consists of several steps to train a model. It is iterative - steps are repeated to continuously improve the accuracy of the model.

Complete Machine Learning Pipeline with Iteration

Step 1: Problem Definition

AspectDetails
GoalDecide what information is needed
MethodsCommittee meetings, brainstorming, analysis of business objectives
OutcomeA project specification for the Data Scientists
Key QuestionsWhat do I have? What do I need? What can I get?

Step 2: Data Collection

AspectDetails
GoalRepresentative sample of the population
MethodsExtract existing data from databases, gather new data
OutcomeA large dataset

Step 3: Feature Extraction

AspectDetails
GoalSpecifying features useful for prediction
MethodsCheck distributions, correlation/covariance checks, drop unique identifiers
OutcomeA "flat" (single table) dataset with selected features

Feature Selection Considerations

  • Flattening: Convert complex data to tabular format
  • Labeling: Assign target values
  • Dimensionality reduction: Remove redundant features

Step 4: Data Preparation

AspectDetails
GoalClean up (normalize, regularize) the data values
MethodsScaling, dummy variables, interpolating missing values
OutcomeNormalized data in numeric form appropriate for modeling

GIGO Principle

"Garbage In, Garbage Out" - If we use unclean data for machine learning, the result will not be satisfying for our end application.

Step 5: Algorithm Selection

AspectDetails
GoalNo a-priori best algorithm; must experiment
MethodsDraw on problem statement, domain knowledge, knowledge of ML and statistical methods
OutcomeA set of algorithms appropriate for the analytical task

Step 6: Performance Evaluation

AspectDetails
GoalTest candidate algorithms and evaluate predictive accuracy
MethodsFit each algorithm on training data, analyze confusion matrix and metrics
OutcomeComparison of results from selected algorithms

Step 7: Model Validation

AspectDetails
GoalEvaluate robustness, check for over-fitting, refine hyperparameters
MethodsGrid search CV, k-fold CV
OutcomeOptimization of objective function, good generalization performance

The Iteration Cycle

The model validation phase leads to new rounds of performance evaluation. The set of candidate algorithms is gradually reduced until one algorithm (or ensemble) is selected.

Essential ML Terminology

TermDefinition
DatasetSample of observations organized into rows and columns
FeatureA column (predictor, variable, attribute)
Training ExampleA row (observation, record, instance)
TargetWhat we predict (label, ground truth)
ModelFunction y = f(x) representing the relationship
ClassifierLearning algorithm + Loss function + Optimizer
Learning AlgorithmInstructions to create a model from training data
Loss FunctionMeasures error for single training example
Cost FunctionAverage loss over entire dataset
OptimizerAlgorithm to minimize cost function
Model ParametersLearned from data (e.g., slope, weights)
HyperparametersSet by programmer (e.g., learning rate, iterations)
FittingProcess of learning model parameters
